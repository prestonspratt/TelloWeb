🚁 DroneCode Deployment Guide

How to upload the DroneCode app to your IONOS website and run the local proxy.

Here's how all the pieces connect when you fly:

DroneCode UI

The UI lives on your IONOS server. The proxy runs locally on your laptop while flying.

1
Understand the file structure
What you downloaded and where each piece goes

Your download contains two parts: the web app (goes to IONOS) and the proxy server (runs on your laptop when flying).

dronecode/
  public/                    ← upload this folder to IONOS
    index.html              ← the DroneCode web app
  server/                     ← keep on your laptop
    proxy.js                 ← the WebSocket↔UDP bridge
    package.json
    start-windows.bat      ← double-click on Windows
    start-mac-linux.sh     ← run on Mac/Linux
2
Upload to IONOS via FTP
Using FileZilla (free) — takes about 5 minutes
2.1

Get your IONOS FTP credentials

Log in to your IONOS account at my.ionos.com. Go to Hosting → Webspace, then click your hosting package. Find the FTP section. Note down:

# You'll need these three things:
FTP Server:  ftp.your-domain.com  (shown in IONOS panel)
Username:    your-ftp-username
Password:    your-ftp-password
2.2

Download and install FileZilla

Download the free FileZilla Client from filezilla-project.org and install it. (Client only — not the Server edition.)

2.3

Connect FileZilla to IONOS

Open FileZilla. In the top bar, fill in:

Host:      ftp.your-domain.com
Username:  your-ftp-username
Password:  your-ftp-password
Port:      21

Click Quickconnect. You'll see your server files appear on the right side.

⚠️If connection fails, try port 22 (SFTP) instead of 21. In IONOS this may require enabling SFTP in your hosting panel first.
2.4

Navigate to your web root

On the right panel in FileZilla (the server side), navigate to your web root folder. For IONOS this is typically:

/htdocs/         ← most common for IONOS shared hosting
/public_html/    ← alternative name
/www/            ← less common

If you see existing files like index.php or a Welcome page, you're in the right place.

💡You can put DroneCode in a subfolder like /htdocs/dronecode/ so it lives at yourdomain.com/dronecode/ — this avoids replacing your existing site.
2.5

Upload index.html

On the left panel (your computer), navigate to the dronecode/public/ folder. Drag index.html from the left panel to the right panel (server). FileZilla will upload it.

✅Once uploaded, visit your domain in a browser. You should see the DroneCode interface! If you put it in a subfolder, visit yourdomain.com/dronecode/
3
Set up the local proxy
This runs on your laptop each time you fly — one-time setup
3.1

Install Node.js

Download and install Node.js from nodejs.org. Choose the LTS version. This is a one-time install.

Verify it worked by opening a Terminal (Mac) or Command Prompt (Windows) and typing:

node --version    # should show v18.x.x or higher
npm --version     # should show 9.x.x or higher
3.2

Install proxy dependencies

Open Terminal/Command Prompt. Navigate to the server/ folder:

# Mac/Linux:
cd ~/Downloads/dronecode/server
./start-mac-linux.sh    # this installs and starts in one step

# Windows:
cd C:\Users\YourName\Downloads\dronecode\server
start-windows.bat       # double-click works too

The first run will install the ws package automatically, then start the proxy.

3.3

4
Fly your first mission
Every time you want to fly, follow these steps in order
4.1

Power on the Tello

Press the power button. Wait for the indicator LED to blink yellow rapidly — this means it's in WiFi hotspot mode and ready.

4.2

Connect your laptop to Tello WiFi

In your WiFi settings, connect to the network named TELLO-XXXXXX (the X's are unique to your drone).

⚠️Your laptop will lose internet access while on Tello WiFi — this is normal. The drone has its own network. You can still use your website since it's already loaded in the browser.
4.3

Start the proxy

Run start-windows.bat or ./start-mac-linux.sh. Wait for it to say "WebSocket server running".

4.4

Open DroneCode and connect

Visit your IONOS-hosted page (e.g. yourdomain.com). Click the "Not Connected" pill in the top-right. The proxy address defaults to localhost:8080 — click Connect.

4.5

Build and run your mission

Drag blocks to build your mission. A basic takeoff-and-return looks like this:

🛫 Takeoff
⬆️ Forward   100 cm
⬆️ Up        50 cm
🏠 Return & Land

Click ▶ Run Mission. The drone will execute each block in sequence. Watch the Console tab for live command responses from the Tello.

🚨Safety: Always fly in a large open area. Keep hands clear of propellers. Have the Tello app on your phone as a backup emergency stop. Keep the drone in visual range at all times.
5
Troubleshooting
Common issues and fixes

❌ "WebSocket error. Is the proxy running?"

The browser can't reach the proxy. Make sure: (1) the proxy terminal is open and shows "WebSocket server running", (2) you're using localhost:8080 in the connect dialog, (3) no firewall is blocking port 8080.

❌ Proxy starts but drone doesn't respond

Make sure your laptop is connected to the TELLO-XXXXXX WiFi network, not your home WiFi. The Tello's IP is always 192.168.10.1 on its own network.

❌ "Port 8080 is already in use"

Another app is using port 8080. Open proxy.js in a text editor, find const WS_PORT = 8080 on line 1, change it to 8081. Also update the port in the DroneCode connect dialog.

❌ Commands send but drone doesn't move

The drone may be in an error state. Land it, power cycle it, reconnect to its WiFi, restart the proxy, and try again. Check that the battery is above 20%.

❌ IONOS page shows "403 Forbidden"

The file was uploaded to the wrong directory, or file permissions are wrong. In FileZilla, right-click index.html on the server → File Permissions → set to 644.
